在发布之前的检查清单

每一个前端项目发布前的检查清单

  1. fix lint 保持代码合乎 规范
  2. run build & Analyzer 检查代码中的依赖项，尽可能的 按需引入
  3. 静态资源开启gzip
  4. 视情况 引入前端错误&性能监控

  5. SEO优化项：

    • 使用 a tag (router-link in vue) 代替 js写的跳转
    • 合理使用html标签，不要全是div
    • 合理设置index.html 中 keywords, description 的内容

  6. 检查meta标签

  7. 升级浏览器的代码

  1. 加入webpack打包时间方便排除缓存错误

  1. 修改favicon.ico

  2. 本地起一个静态服，使用chrome edge ie11，测试打好包的代码， 如果有单元测试可以跑一下

  3. 在readme中补充发布，变更记录
